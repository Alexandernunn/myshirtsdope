Fix the color image display on the Product Detail page. The current product 
data has colorImages: null for all products. We need to fetch and map 
color-specific preview images from the Printful API at the variant level.

---

🔧 THE PROBLEM
The product data being stored/fetched only captures the default imageUrl 
and lists color names, but does NOT fetch the per-variant preview images 
from Printful. The colorImages field is null on every product.

---

🔧 FIX — FETCH VARIANT-LEVEL IMAGES FROM PRINTFUL

When fetching a single product detail (GET /store/products/{id}), Printful 
returns a sync_variants array. Each variant object contains:

- variant_id
- name (includes color and size like "Black / S")
- color (the color name)
- files[] array — inside this, find the file with type: "preview" which has 
  a preview_url — this is the color-specific product mockup image

Update the get-product Netlify Function to:

1. Fetch GET https://api.printful.com/store/products/{printfulId}
2. From the response, loop through result.sync_variants
3. For each variant, extract:
   - The color name from variant.name (split on " / " and take the first part)
   - OR from variant.product.color if available
   - The preview image from variant.files.find(f => f.type === "preview")?.preview_url
4. Build a colorImages map object:
   {
     "Black": "https://files.cdn.printful.com/...",
     "Red": "https://files.cdn.printful.com/...",
     "True Royal": "https://files.cdn.printful.com/...",
     ...
   }
   Only add a color to the map the FIRST time you see it (since multiple 
   sizes share the same color, you only need one image per color)
5. Return this colorImages map as part of the product detail response
